# Basic GIT learnings 

## What is GIT?

### Git is a version control system that helps in tracking changes made in code, collaborating with other and reverting back to previous versions in case of failure of updates.

## Core concepts:

-	Repository: folder where projects are stored
-	Working directory: actual project files
-	Staging area: the preparation zone before saving changes
-	Commit: a checkpoint of changes made
-	Basic Commands to initialise a repo, add files commit changes, check status add files to staging and connect to remote
-	Branching: area to work aside from main code
-	Merging: commands to merge a branch to main

## Summary:
### Git helps in collaboration with teams to build scalable applications and maintaining it. Adding features and making updates to features becomes easy using the branching concept as it lets to work out the code without interfering with the already working code. In case of failure of any feature or bugs in a new added feature GIT allows to revert back to the old versions, making development hassle free and the application not being taken down completely.
